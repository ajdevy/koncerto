# Architecture Design Task

**Purpose:** Create technical architecture and design decisions

## When to Use
- After PRD is approved
- Before implementation begins
- When designing system components
- For technical decision making

## Process

1. **Review PRD**
   - Understand requirements
   - Identify technical constraints
   - Review non-functional requirements

2. **Design Architecture**
   - Define system components
   - Design module structure
   - Create data flow diagrams
   - Define APIs

3. **Make Technical Decisions**
   - Technology stack
   - Design patterns
   - Integration approaches
   - Security considerations

4. **Document**
   - Create Architecture Document
   - Define module responsibilities
   - Specify interfaces
   - Document decisions

5. **Review**
   - Technical review with team
   - Validate against requirements
   - Get approval

## Output Format

```markdown
# Architecture Document: [Project Name]

## 1. System Overview
[High-level description]

## 2. Technology Stack
| Layer | Technology | Version |
|-------|------------|---------|
| Language | [Language] | [Version] |

## 3. Module Architecture
### Module Dependency Graph
[Visual representation]

### Module Responsibilities
| Module | Responsibility | Key Classes |
|--------|---------------|-------------|
| [Module] | [Responsibility] | [Classes] |

## 4. Data Flow
[Sequence diagrams or flowcharts]

## 5. APIs
| Endpoint | Method | Description |
|----------|--------|-------------|
| [Path] | [Method] | [Description] |

## 6. Data Models
[Entity relationship diagrams]

## 7. Security Considerations
[Security architecture]

[Performance targets and approach]

## 9. Deployment
[Deployment architecture]

## 10. Technical Decisions
| Decision | Options Considered | Choice | Rationale |
|----------|-------------------|--------|-----------|
| [Decision] | [Options] | [Choice] | [Why] |
```
