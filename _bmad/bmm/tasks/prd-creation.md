# PRD Creation Task

**Purpose:** Create a comprehensive Product Requirements Document

## When to Use
- After Project Brief is approved
- Before architecture design
- When defining new features
- For stakeholder alignment

## Process

1. **Review Project Brief**
   - Understand vision and goals
   - Identify target users
   - Review success metrics

2. **Define Requirements**
   - Functional requirements (FR)
   - Non-functional requirements (NFR)
   - Constraints and assumptions

3. **Create Epics**
   - Break product into major features
   - Define epic goals
   - Estimate complexity

4. **Write User Stories**
   - Follow standard format
   - Include acceptance criteria
   - Estimate story points

5. **Validate**
   - Review with stakeholders
   - Ensure completeness
   - Get approval

## Output Format

```markdown
# Product Requirements Document: [Project Name]

## 1. Executive Summary
[2-3 sentence overview]

## 2. Problem Statement
[Clear problem definition]

## 3. Target Users
[User personas and needs]

## 4. Goals & Success Metrics
[Measurable objectives]

## 5. Functional Requirements
| ID | Requirement | Priority |
|----|-------------|----------|
| FR-01 | [Requirement] | P0 |

## 6. Non-Functional Requirements
| ID | Requirement | Target |
|----|-------------|--------|
| NFR-01 | [Requirement] | [Target] |

## 7. Epics & Stories
### Epic 1: [Name]
#### Story 1.1: [Title]
- **As a** [user type]
- **I want** [goal]
- **So that** [benefit]
- **Acceptance Criteria:** [Criteria]

## 8. Out of Scope
[What's not included]

## 9. Open Questions
[Unresolved items]
```
